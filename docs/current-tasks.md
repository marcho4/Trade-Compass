## Дополнительные данные для Ticker Screen

Новый дизайн экрана тикера требует данных, которых сейчас нет в API. Во фронте эти поля заглушены прочерками — нужно постепенно реализовать источники.

1. **Идентификация бумаги** (financial-data):
   - `ISIN` по тикеру
   - `industry` (под-отрасль внутри сектора)
   - `freeFloat` (%)
   - `lotSize` уже есть — вывести в интерфейс при необходимости
2. **Рыночные коэффициенты** (financial-data, считать от raw-data + price):
   - `P/E`, `P/B`, `P/S`
   - `Div Yield` (годовая дивдоходность по последним выплатам)
   - `Beta` (к IMOEX за 1 год)
   - 52W Low / High — уже считается в `use-ticker-screen-data`, но можно вынести в API, чтобы не тянуть 365 свечей
3. **Консенсус аналитиков** (новый источник):
   - рекомендация (BUY / HOLD / SELL) и распределение (Str Buy / Buy / Hold / Sell / Str Sell)
   - 12M target price, upside, дата последнего апдейта, количество покрытий
4. **Bank-specific показатели** для финтех-сектора (ROE, CIR, NPL, CAR, LCR):
   - сейчас в raw-data есть только базовые поля (netInterestIncome, commissionIncome, creditLossProvision)
   - нужно либо считать производные показатели в API, либо расширить raw-data
5. **Котировки внутри дня** — свечи с `interval=1` уже возвращают open/high/low/volume/value,
   но для «Оборот» и «Объём» удобнее сделать отдельную ручку `/session-stats?ticker=...`,
   чтобы не тянуть сотни свечей.
