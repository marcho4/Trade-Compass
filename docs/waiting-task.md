# система ожидания задач для генерации сценариев

Парсер парсит сайт, считает кол-во отчетов N, которое удалось успешно обработать и затем отправляет сообщение в ai-service на ожидание N - delta сообщений raw data results, где delta - конфигурируемое значение. 
(я хочу сделать 1, если потеряется один raw-data на глобальную картину это не прям сильно повлияет)

При получении сообщения от парсера, ai-servise должен инкрементировать значение pending_count по ключу (task_id, raw-data) на 1

При создании компании сразу отправляется risk-and-growth-expect в ai-service, который при получении сообщения должен увеличить на 1 pending_count по ключу (task_id, risk-and-growth).

ai-service при успешном извлечении raw data должен отправить raw-data-success сообщение по id. При получении этого сообщения в табличке tasks pending_count у task_id должен уменьшиться на 1 для типа raw-data

ai-service при успешном извлечении Risk-and-Growth должен отправить risk-and-growth-success по id. При получении этого сообщения в табличке tasks pending_count у task_id должен уменьшиться на 1 для типа risk-and-growth.

В фоне в ai-service всегда будет крутиться горутина, которая время от времени будет ходить в табличку текущих тасков, находить там таски, у которых pending_count = 0 и отправлять задачу на scenarios-generate в свою же очередь.
