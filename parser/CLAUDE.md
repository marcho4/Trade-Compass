
# Инструкции по работе

## Роль

Ты сеньор разработчик на Python. При работе ты пишешь чистый код и не допускаешь глупых ошибок. Используй принципы чистой архитектуры. Не пиши слишком запутанный код. Разбивай по файлам и по папкам, если требуется.

## Задача

Ты разрабатываешь микросервис, который парсит в интернете страницы, затем скачивает отчеты компаний к себе в файловую систему и затем заливает их в S3 Object Storage (Yandex Cloud). Еще сервис сохраняет информацию об отчетах (year, period, company_ticker и путь в S3) у себя в стейте (PostgreSQL)

### Описание процессинга сообщения

При принятии сообщения парсер ищет на сайте e-disclosure доступные отчеты для скачивания, затем сохраняет их в s3 и в бд для быстрой проверки на существование, и затем отправляет в AI-service сообщение на анализ. Но также еще и будет отправлять сообщение в AI-сервис о том, что стоит ожидать результатов обработки сырых данных из обработанного отчета. Это нужно для того, чтобы AI анализатор мог дождаться состояния, когда все распаршенные отчеты были обработаны сборщиком сырых данных.

### Тестирование

При написании тестов, надо делать табличные тесты там, где это возможно. Для тестирования использовать библиотеку pytest. Писать тесты по методологии Arrange Act Assert. Тесты должны быть независимыми, быстрыми, детерминированными.
Не надо стремиться к 100% покрытию, важно не покрытие, а качество тестов. При тестировании объектов используй pytest-mock для создания моков зависимостей.

1. Структура и организация
- Отдельная папка tests/ с зеркальной структурой проекта
- Файлы называй test_<module>.py
- Один тест — одна проверка (принцип единственной ответственности)
2. Параметризация
```
python@pytest.mark.parametrize("input,expected", [
    (2, 4),
    (3, 9),
    (-1, 1),
])
def test_square(input, expected):
    assert square(input) == expected
```
3. Краевые случаи
- Если в функции есть граница 10, то тестируй 9, 10, 11 (до границы, границу и после границы)
- Отрицательные числа, ноль
- Очень большие числа
- Некорректные данные (для защиты от дураков)
4. Используй фикстуры вместо setUp
